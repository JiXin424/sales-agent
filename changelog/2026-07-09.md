## Memory-Eval Operations: 采样 trace + 提升 regression 表（Spec 4 Task 18）

- **改动对象**：`src/sales_agent/models/memory_eval.py`（新增）、`src/sales_agent/models/__init__.py`、`src/sales_agent/migrations/versions/0015_memory_eval_operations.py`（新增）、`tests/integration/test_memory_eval_models.py`（新增）。
- **类型**：DB schema（Alembic migration 0015）+ ORM 模型 + 集成测试。
- **影响范围**：新增两张表，为 memory-eval 运营 harness（Spec 4 §8/§9）持久化「采样生产 trace」与「提升后的匿名 regression 场景」。无运行时行为变更，不触发任何流量路径。
- **改动明细**：
  - **`memory_eval_traces`（`MemoryEvalTraceRecord`）**：受限保留的采样生产 trace。列：`id/tenant_id/scope_hash/thread_id/trace_json(JSON)/retention(default 'restricted')/status(default 'sampled')/captured_at(timestamptz)/created_at/updated_at`。索引：`ix_memory_eval_traces_tenant_id`、`ix_memory_eval_traces_tenant_status(tenant_id,status)`。标识符保护：scope 走 `scope_hash`（哈希），无明文租户/用户 id。
  - **`promoted_regressions`（`PromotedRegression`）**：从 trace 提升的匿名 regression 场景。列：`id/tenant_id/source_trace_id/scenario_json(JSON)/status(default 'draft', draft→reviewed→committed)/anonymized(default true)/review_notes/created_at/updated_at`。索引：`ix_promoted_regressions_tenant_id`。
  - 模型在 `models/__init__.py` 第 81 行注册（`from .memory_eval import MemoryEvalTraceRecord, PromotedRegression`），并加入 `__all__`，供 Alembic `env.py` 的 metadata 发现。
  - Migration `0015_memory_eval_operations`：`down_revision = "0014_user_memory_profiles"`，仅 `create_table` 两张表（不混 add_column），遵循 0014 命名规范（`Text` 列、显式 `PrimaryKeyConstraint`、独立 `create_index`、时间戳列用 `Text + server_default=func.now()` 与 `TimestampMixin` 一致）。
- **原因**：Task 18 为 23 任务计划的存储层基础；后续 online-sample（Task 19）、promote-trace（Task 21）等模式需要这两张表落库。
- **验证**：集成测试 `tests/integration/test_memory_eval_models.py`（2 项）经 `db_session` fixture（`Base.metadata.create_all`）全绿，证明模型正确（primary gate）。Migration 单独验证：测试库 `create_all` 建全表 → drop 两张新表 → `alembic stamp 0014_user_memory_profiles` → `alembic upgrade head` 仅应用 0015 成功，表结构与模型一致（列类型/默认/索引核对通过），测试库到 head `0015_memory_eval_operations`。

## 记忆评估与生产运维套件（Spec 4 整体交付）

- **改动对象**：`eval/memory_eval/`（新增整套包）、`src/sales_agent/services/memory_eval_trace.py`、`src/sales_agent/models/memory_eval.py`、迁移 `0015_memory_eval_operations.py`、`scripts/run_memory_eval_gate.sh`（新增）、`docs/runbooks/memory-evaluation.md`（新增）、`README.md`（更新日志行）、`changelog/2026-07-09.md`。
- **类型**：新功能（离线评估 + 发布门禁 + 生产运维）+ 运维文档。
- **影响范围**：离线评估、发布门禁、生产在线采样；**不改动在线 Graph 请求路径**，无运行时行为变更（除新增的 online-sample/promote-trace 模式独立落库）。所有评估产物落在 `eval/` 与两张新表，生产流量零侵入。
- **改动明细**：
  - **统一多轮场景 schema（§4）**：`MultiturnScenario` 版本化数据集 `eval/memory/datasets/multiturn_v1.jsonl`（42 场景，覆盖 dingtalk-staging 标签），denominator/版本随报告暴露。
  - **版本化报告（§7）**：`EvaluationReport` + `MetricResult`（含 applicability/denominators/errors/versions），`compare` 模式生成 baseline↔candidate 对照。
  - **确定性指标四组（§5）**：isolation/safety（fail-closed）、turn/topic（含 clarification completion、restore/restart recovery）、long-term-memory、recall-profile/turn-relation 等语义评估器。
  - **确定性 model double + scenario runner 驱动真实 `invoke_online_turn`（§3.2）**：`graph-multiturn` 用 stub model 跑真实 Online Graph + PG，端到端可复现。
  - **七个 CLI 模式（§11）**：`unit-memory` / `graph-multiturn` / `model-multiturn`（绑真实 `resolve_online_models`）/ `dingtalk-staging` / `compare` / `online-sample`（非阻塞、bounded `--max-threads`）/ `promote-trace`，统一经 `eval/memory_eval/runner.py` 派发。
  - **fail-closed 发布门禁（§6）**：隔离/安全门禁零跨租户/agent/user 泄漏、零禁止记忆写入、零 assistant-output→user-memory 激活；阈值改动需版本化 rationale，绝不降阈值过关。
  - **在线采样（§3.5）**：`online-sample` best-effort、bounded，不阻塞用户流量，受限保留写入 `memory_eval_traces`（scope 走 `scope_hash` 哈希，无明文标识符）。
  - **promote-trace 反馈闭环（§9）**：生产失败 trace → 匿名 `MultiturnScenario`（`status=draft`）→ 评审 `reviewed` → `model-multiturn` 复现 → 提交数据集；落 `promoted_regressions` 表。
  - **运维入口与文档**：`scripts/run_memory_eval_gate.sh`（refuse 非 `*test*` DB；Phase A unit-memory + Phase B graph-multiturn + Phase C 数据集覆盖/门禁）；`docs/runbooks/memory-evaluation.md` 操作手册（门禁、七模式、发布门禁、promote-trace 工作流）；README `## 更新日志` 新增本条索引行。
- **原因**：Spec 4 — 让多轮与记忆质量在发布前可度量、发布后可维护；补齐 23 任务计划的运维入口与强制文档（repo 规则：changelog + README 更新日志行）。
- **验证**：`bash -n scripts/run_memory_eval_gate.sh`（语法零错）；`PYTHONPATH=src:. python3 -m pytest tests/unit/eval tests/unit/services/test_memory_eval_trace.py -q` 全绿（覆盖 schema/dataset/报告/指标/门禁/采样/promote/runner 派发/model double/trace）；可选真实门禁 `TEST_DATABASE_URL=postgresql+asyncpg://sales_agent:sales_agent_dev@172.26.0.4:5432/sales_agent_test ./scripts/run_memory_eval_gate.sh`。
