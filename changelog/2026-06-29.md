# 2026-06-29 变更日志

## models.json — 多模型配置

- **对象**: `models.json`（新增）
- **类型**: 配置
- **影响范围**: 全局模型配置
- **明细**: 新增项目根目录 `models.json`，集中管理多模型/多 API 配置。每个模型自包含 `base_url`、`api_key_env`（环境变量名，非密钥值）、`embedding_model` 等参数。支持 `default_model` 指定默认模型。文件可安全提交 git。
- **原因**: 原有模型配置散落在 `.env` / `config/default.yaml` / 代码硬编码中，且只支持单一模型。统一配置后评估脚本和 app 共用同一份配置。

## model_registry.py — 模型注册表

- **对象**: `src/sales_agent/core/model_registry.py`（新增）
- **类型**: 功能
- **影响范围**: 模型加载与查找
- **明细**: 新增 `ModelRegistry` 类，从 `models.json` 加载所有模型配置，解析环境变量获取实际 API key。提供 `get(name)` 按名查找和 `list_models()` 列出所有模型。
- **原因**: 为 `tenant_runtime` 和评估脚本提供统一的模型配置加载入口。

## tenant_runtime.py — 集成 models.json

- **对象**: `src/sales_agent/core/tenant_runtime.py`
- **类型**: 增强
- **影响范围**: app 启动时的模型初始化
- **明细**: `from_current_env()` 优先从 `models.json` 加载模型配置；文件不存在时回退到原有环境变量方式，保证向后兼容。
- **原因**: 让 app 启动时自动使用 `models.json` 的 `default_model`。

## 请求级模型切换

- **对象**: `api/schemas.py`, `api/routes/agent.py`, `services/chat_pipeline.py`
- **类型**: 增强
- **影响范围**: Chat API
- **明细**: `ChatRequest` 新增可选 `model` 字段；agent route 将 `model` 传递给 pipeline；pipeline 的 `execute()` 方法在收到非空 model 且与默认模型不同时，从 registry 获取目标模型配置，构建独立的 `OpenAICompatibleChat` 实例（embedding 保持不变）。评估脚本通过此机制在一个 app 实例上测试多个模型。
- **原因**: 支持评估脚本在同一 app 实例上切换模型。

## Dockerfile — 包含 models.json

- **对象**: `Dockerfile`
- **类型**: 配置
- **影响范围**: Docker 镜像构建
- **明细**: COPY 步骤增加 `models.json`。
- **原因**: 确保 models.json 打进镜像。

## 评估脚本

- **对象**: `eval/run_conversation_eval.py`（新增）, `eval/questions.md`（新增）, `eval/merge_results.py`（新增）
- **类型**: 功能
- **影响范围**: 评估体系
- **明细**:
  - `run_conversation_eval.py`: 端到端对话评估脚本。读取问题文件（支持 .md / .json），对每个（模型, 问题）组合调用 `POST /agent/chat`（走完整链路），输出 JSON + Markdown 报告。
  - `questions.md`: 30 道销售场景评估问题。
  - `merge_results.py`: 合并多个评估 JSON 结果文件（分布式场景）。
- **原因**: 建立自动化评估体系，支持多模型对比和多服务器并行评估。

## CI/CD workflow

- **对象**: `.gitea/workflows/eval.yml`（新增）
- **类型**: CI/CD
- **影响范围**: 持续集成
- **明细**: Gitea Actions workflow。PR 修改核心模块时自动触发评估，也支持手动触发。启动 PostgreSQL + app → 运行评估脚本 → 上传结果 artifact。API keys 通过 Gitea Secrets 注入。
- **原因**: 将评估集成到 CI/CD 流程中，每次变更自动验证模型表现。
