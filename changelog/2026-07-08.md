# 2026-07-08 升级日志

## 场景教练（Scenario Coach）新增

- **改动对象**：`src/sales_agent/scenarios/`（新包：`data/销冠智慧教练手册.md` + `models.py` + `loader.py` + `matcher.py` + `prompt.py`）、`graph/online/{nodes,edges,graph,state}.py`、`graph/node_metadata.py`、`core/config.py`、`integrations/dingtalk/{graph_stream,citation}.py`、`services/online_conversation.py`、`config/default.yaml`、`.env.example`。
- **类型**：新功能（feature flag，默认关闭）。
- **影响范围**：Online Graph 新增 `scenario_coach` 拦截节点 + `log_scenario_response` 落库节点；仅在 `SCENARIO_COACH_ENABLED=true` 的实例生效，其余实例行为零变化（拓扑在开关关闭时与改造前完全一致，176 个图单测全过）。
- **改动明细**：
  - 新增自包含包 `scenarios/`：`data/销冠智慧教练手册.md`（6 场景 11 预设问题，随镜像分发，无 DB）、`loader.py`（md→`ScenarioRegistry` 进程级单例，fail-open：解析失败降级为关闭不 crash stream）、`matcher.py`（LLM 11 选 1 + none 分类，复用 `parse_model_json`，置信度阈值默认 0.8，失败 fail-open 放行）、`prompt.py`（system + 2 few-shot + 11 题枚举 + JSON schema）、`models.py`（`Scenario`/`ScenarioQuestion`/`AnswerSection`/`ScenarioMatchDecision`）。
  - Online Graph 在 `normalize_turn` 后插入 `scenario_coach` 节点：命中→返回预设 `answer_dict`（含 `《销冠智慧教练手册·2026年4月版》` 来源，`source_type=scenario_coach`→`教练手册` 标签）+ `response_kind="scenario"`→`log_scenario_response`→END（跳过 context_resolution/evidence_routing/retrieval/generation）；未命中→透传原路径（`chat`→context_resolution / `direct_chat`→direct_evidence_routing）。
  - 新增 `ScenarioCoachConfig{enabled=False, confidence_threshold=0.8}`，env 覆盖 `SCENARIO_COACH_ENABLED`（bool）/ `SCENARIO_COACH_CONFIDENCE_THRESHOLD`（float，**repo 首个 float-from-env 覆盖**）。
  - `scenario_coach_enabled` 状态字段注入**两个** input_state 构造点（`online_conversation.py` HTTP 路径 + `graph_stream.py` 钉钉 stream 生产路径）——覆盖 lessons #35 的 stream 入口陷阱；`route_online_message` 仅在开启且 `flow_action∈{chat,direct_chat}` 时改路到 `scenario_coach`，其余（duplicate/guided_flow）不变。
  - `citation.py` 新增 `scenario_coach → 教练手册` 来源标签；`graph/node_metadata.py` 补 2 节点元数据（lessons #34：graph_debug 用集中映射表识别节点）。
  - 无 DB 变更、无 Alembic migration、无 seed；场景内容随镜像分发，部署到任意开了 flag 的实例即自动获得同样 6 场景 11 问题。
- **原因**：针对单租户实例提供预设销售场景问答能力，命中即出预设答案（含手册来源），未命中走正常 AI；通过 env flag 实现实例级开关（本机两实例 A 开 B 关）与跨机（CI/CD prod3）快速部署。
- **验证**：`.venv/bin/pytest tests/unit -q` 非依赖 DB 的 944 项全绿（含 176 图单测 + 全部 scenario_coach 单测/集成测）；唯一失败是 `ontology/test_extractor` 与 topic_model/ingestion 等 97 项依赖运行中 Postgres(5432) 的测试在无 DB 的 worktree 环境报 `Errno 111 Connect failed`，与本改动无关（scenario_coach 零 DB）。生产验证（lessons #4）待部署后查 `docker logs sales-agent-<tenant>-stream` 确认 `scenario_coach loaded 6 scenarios / 11 questions` 且无 crash。
