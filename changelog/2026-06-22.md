# 2026-06-22 升级日志

## Prompt 全层解耦到 DB 版本管理 + 访前/访后卡片 bug 修复

### 概述
将 sales-agent **所有层 prompt**（task / system / router / risk / coach）统一纳入 DB 版本管理，
运营可在管理后台改任意层 prompt 即时生效（主链路 + 钉钉 + CLI），无需改代码重部署。同时修复
访前/访后卡片模板的双花括号 bug。

此前仅主 Web 链路（`ChatPipeline`）接 DB，钉钉流式（主要生产渠道）和 CLI 直接用代码常量，
运营改后台对这两条链路不生效；system/router/risk/coach 四类 prompt 焊死在代码里。

---

### 改动明细

#### 1. Bug 修复：访前/访后卡片用户输入未注入 LLM
- **对象**：`prompts/visit_preparation.py`、`prompts/post_visit_review.py`
- **类型**：bug 修复
- **影响范围**：访前作战卡、访后机会推进卡（Web `/agent/chat` 链路）
- **明细**：模板「用户输入」区误用双花括号 `{{message}}`，而 `agent_executor` 用 `str.format()` 填充，
  Python 把 `{{` 当转义、输出字面 `{message}`，导致用户输入/上下文/检索内容**未注入 LLM**。改为
  单花括号（JSON 示例区的双花括号保留）。
- **原因**：这两个卡片任务此前从未走过 `.format()`（钉钉走 `quick_session` 绕过），所以 bug 未暴露。
- **测试**：新增 `tests/unit/test_visit_post_visit_placeholders.py`（6 项）。

#### 2. 引入 Alembic 迁移机制
- **对象**：`alembic.ini`、`src/sales_agent/migrations/`（`env.py` + `0001_baseline` + `0002_prompt_category`）
- **类型**：基础设施
- **影响范围**：数据库 schema 变更流程
- **明细**：项目此前用 `create_all` 自动建表（无法给已有表加列）。引入 Alembic（async engine，
  URL 从 settings 注入）。`0002_prompt_category` 给 `prompt_versions` 加 `prompt_category`/`prompt_key`/
  `required_placeholders_json` 列、`task_type` 改 nullable、加复合索引、回填历史数据。upgrade/downgrade
  往返验证通过。
- **原因**：CLAUDE.md 要求 DB 变更走 Alembic；本次加列需要正式迁移。

#### 3. PromptVersion 数据模型扩展
- **对象**：`models/prompt.py`
- **类型**：数据模型
- **明细**：加 `prompt_category`（task|system|router|risk|coach）、`prompt_key`、
  `required_placeholders_json`；`task_type` 改 nullable；加复合索引。
  `AgentPromptSet.task_prompt_versions_json` schema 升级为嵌套 `{category: {key: version_id}}`，
  registry 读取兼容旧扁平格式。

#### 4. Registry 通用化 + 四链路接入
- **对象**：`services/prompt_defaults.py`（新）、`prompt_registry.py`、`prompt_resolver_helper.py`（新）、
  `agent_executor.py`、`task_router.py`、`risk_checker.py`、`chat_pipeline.py`、
  `integrations/dingtalk/streaming_handler.py`、`cli.py`、`api/routes/prompts.py`、`api/schemas.py`
- **类型**：架构
- **明细**：
  - `prompt_defaults.py` 集中 22 个内置 prompt（12 task + system + router + risk + 7 coach）为 single source of truth。
  - `PromptRegistry.resolve_prompt(category, key, tenant_id, agent_id)` 通用三级回退（Agent 绑定 → tenant active → 内置默认）。`COALESCE(prompt_key, task_type)` 兼容旧行。
  - `agent_executor` 加 `system_prompt_text` 参数（system prompt 也走 DB）。
  - 公共 helper `resolve_execution_prompts` / `resolve_risk_prompt` / `resolve_quick_session_prompts`。
  - **钉钉流式 + CLI 此前绕过 registry（直接用代码常量），现经 helper 接入 DB 版本管理。**
  - `task_router`/`risk_checker` 支持调用方注入解析后的 prompt。
  - preview 端点通用化（按 category 渲染 router/risk/coach），新增 `GET /prompts/builtin`。
- **原因**：钉钉是主要生产渠道却绕过 DB；system/router/risk/coach 焊死代码。

#### 5. Coach prompt 接入
- **对象**：`prompts/coach_quick.py`（新）、`coach/daily_evaluator.py`、`coach/quick_session.py`
- **类型**：架构
- **明细**：`daily_evaluator._call_llm` 经 registry 解析 `coach_daily_eval` + system 消息；
  `quick_session` 三个 f-string prompt 改写为 `.format` 模板常量（移到 `prompts/coach_quick.py`），
  调用链透传解析的 5 个 prompt。

#### 6. 前端：Agent 级 Prompt 绑定 UI + category 维度
- **对象**：`api/routes/agents.py`、`console/src/api/{types,prompts,agents}.ts`、
  `console/src/utils/constants.ts`、`console/src/pages/Agents/AgentPromptsPage.tsx`、
  `console/src/pages/Prompts/PromptEditPage.tsx`
- **类型**：前端
- **明细**：
  - 后端新增 `PUT /agents/{id}/prompts/bindings/{category}/{key}`（绑定/解绑），自动创建 `AgentPromptSet`。
  - `AgentPromptsPage` 从**只读改为可编辑**：按 category 分 Tabs，每个 prompt_key 支持切换版本（Modal Select）/解绑。
  - `PromptEditPage` 加 `prompt_category` 选择（支持创建 system/router/risk/coach 版本）+ 占位符提示。
  - 前端 `tsc -b` 类型检查通过。
- **原因**：Agent 级 prompt 绑定后端逻辑此前已存在但无运营入口；解耦需闭环。
