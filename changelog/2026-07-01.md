# 2026-07-01 — LangGraph P0-P3 特性集成

## 概述

全面集成 LangGraph 高级特性（P0-P3 优先级），涵盖人机协同(HITL)、跨会话记忆、自定义流式事件、节点缓存、工具注入、编译期中断等 10 项功能。

## P0 — 核心特性

### 1. `interrupt()` + `Command` — 人机协同 (HITL)
- **改动对象**: `graph/nodes/risk_check.py`
- **类型**: 功能升级
- **影响范围**: 高风险回复审核流程
- **改动明细**:
  - 新增 `_AUTO_APPROVE_HIGH_RISK = True` 开关（当前全部放行）
  - `risk_action == "block"` 时调用 `interrupt()` 暂停图执行
  - 自动审批模式下 block 降级为 warn，带安全提示放行
  - 关闭自动审批后，外部通过 `Command(resume=...)` 恢复执行
- **原因**: 为 Pilot 人工审核流程预留基础设施，当前阶段不做实际拦截

### 2. `BaseStore` — 跨会话持久化记忆
- **改动对象**: `graph/checkpoints.py`, `graph/chat_graph.py`, `graph/react_agent.py`, `api/routes/agent.py`, `integrations/dingtalk/graph_stream.py`
- **类型**: 功能升级
- **影响范围**: 全局图执行
- **改动明细**:
  - 新增 `get_store_sync()` / `get_store()` 工厂函数
  - 支持 `InMemoryStore`（开发）和 `AsyncPostgresStore`（生产）
  - `build_chat_graph_compiled` 接受 `store` 参数
  - HTTP 和 DingTalk 路径均传入 Store
- **原因**: 跨会话记忆（用户画像、知识缺口追踪、客户偏好）

## P1 — 高价值特性

### 3. `stream_mode="custom"` + `StreamWriter`
- **改动对象**: `graph/nodes/generation.py`, `graph/nodes/retrieval.py`, `integrations/dingtalk/graph_stream.py`
- **类型**: 功能升级
- **影响范围**: 钉钉卡片流式进度展示
- **改动明细**:
  - `generate_node` 发射 `generation_started` / `generation_executing` / `generation_complete` 事件
  - `retrieve_node` 发射 `retrieval_started` / `retrieval_complete` / `retrieval_skipped` 事件
  - DingTalk stream 通道新增 `custom` 模式消费，卡片更新检索/生成进度
- **原因**: 提升钉钉用户等待体验，可视化处理进度

### 4. `CachePolicy` — 节点级缓存
- **改动对象**: `graph/retry_policies.py`, `graph/chat_graph.py`
- **类型**: 性能优化
- **影响范围**: 检索和生成节点
- **改动明细**:
  - 新增 `CACHE_RETRIEVAL`（TTL=300s）和 `CACHE_GENERATION`（TTL=120s）
  - `retrieve` 和 `generate` 节点添加 `cache_policy` 参数
- **原因**: 减少重复 RAG 调用和 LLM 生成，降低延迟和成本

### 5. `InjectedState` / `InjectedStore` — 工具状态注入
- **改动对象**: `graph/tools.py`
- **类型**: 功能升级
- **影响范围**: ReAct Agent 工具调用
- **改动明细**:
  - `search_knowledge_base` 通过 `InjectedState` 获取 tenant_id
  - `check_customer_history` 通过 `InjectedStore` 读取跨会话客户数据
  - `get_sales_script_template` 通过 `InjectedState` 获取 task_type
- **原因**: 工具可访问图状态和持久化存储，无需手动传参

## 第二批：补充 6 项 LangGraph 特性 (同日第二轮)

### 1. `Send` — 并行 Fan-out 检索
- **改动对象**: `edges/path_conditions.py`, `state.py`, `nodes/generation.py`
- **类型**: 功能升级
- **影响范围**: 检索阶段
- **改动明细**:
  - `sources` 字段添加 `Annotated[list[dict], add]` reducer，支持多方并发写入合并
  - `select_retrieval_path` 新增 `parallel_enabled` 分支：当配置开启时返回 `[Send("retrieve", {path:"ontology"}), Send("retrieve", {path:"rag"})]`
  - `generate_node` 新增 `fan_out_answer_present` 检测：无论哪个 Send 先/后结束，只要已有 `answer_dict.summary` 就跳过生成
- **原因**: 知识问答同时查 Neo4j 和向量库，结果自动合并
- **启用方式**: `config/default.yaml` 中 `retrieval.parallel_enabled: true`

### 2. `ValidationNode` — 工具输出校验
- **改动对象**: `tools.py`
- **类型**: 质量保障
- **影响范围**: ReAct Agent 工具调用
- **改动明细**: 工具签名新增 `ToolRuntime` 参数（`tool_call_id` 注入），校验失败可拦截重试
- **原因**: 防止工具返回格式异常导致 Agent 推理崩溃

### 3. `ToolRuntime` — 工具专属上下文
- **改动对象**: `tools.py`
- **类型**: 功能升级
- **影响范围**: 3 个工具
- **改动明细**: `search_knowledge_base`、`check_customer_history`、`get_sales_script_template` 均接受 `runtime: ToolRuntime` 参数，可访问 `config`、`state`、`tool_call_id`
- **原因**: 工具可获得当前 LLM 调用上下文用于日志/追踪

### 4. `TAG_HIDDEN` — 隐藏 Tracing 噪音
- **改动对象**: `chat_graph.py`
- **类型**: 可观测性
- **影响范围**: LangSmith/LangFuse tracing
- **改动明细**: `validate`、`resolve_tenant`、`load_context`、`log` 节点添加 `tags=[TAG_HIDDEN]`
- **原因**: 基础设施节点对业务分析无价值，隐藏后减少 tracing 噪音

### 5. `Durability` — 检查点写入策略
- **改动对象**: `chat_graph.py`, `graph_stream.py`, `api/routes/agent.py`
- **类型**: 性能优化
- **影响范围**: 全局检查点写入
- **改动明细**:
  - DingTalk 流式路径：`astream(... durability="async")` — 异步写检查点，不阻塞下一步
  - HTTP 路径：`ainvoke(... durability="sync")` — 同步写，保证数据安全
- **原因**: 钉钉流式场景延迟敏感，async 模式减少检查点写入等待

### 6. `REMOVE_ALL_MESSAGES` — 清空对话历史
- **改动对象**: `nodes/fast_commands.py`
- **类型**: 功能完善
- **影响范围**: "新话题" / `/reset` 命令
- **改动明细**: reset 命令现在返回 `history_messages: REMOVE_ALL_MESSAGES`，完全清空消息列表而非只换 `conversation_id`
- **原因**: 之前只换 conversation_id 会导致 stale 历史在 checkpointer 中残留

- **影响范围**: CI 流程

### 使用方式
```bash
# 本地：启动某租户的 Studio
docker compose --profile studio up -d taishan-studio
# 访问：https://smith.langchain.com/studio/?baseUrl=http://localhost:8013

# 生产：每个租户自动生成 studio 服务
python scripts/render-multitenant-deploy.py deploy/tenants.json
docker compose -f docker-compose.generated.yml up -d
# Studio 端口：api_port + 10 (e.g. taishan api:8003 → studio:8013)
```
