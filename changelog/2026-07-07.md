## reactflow 图节点加 Handle 修复「连线不渲染」

### 改动对象与原因
图调试页 reactflow 重画后，节点之间的连线（edge）完全不渲染（DOM 0 条），节点 measured 字段为空。用 puppeteer 抓 console 发现 reactflow error #008：`Couldn't create edge for source handle id: "undefined"`——每条 edge 都因找不到 source handle 创建失败。

根因：reactflow 11 的自定义节点必须渲染 `<Handle>` 组件作为连接锚点，否则 edge 无法 attach。`GraphNode.tsx` 只渲染了卡片 div，没有 Handle，导致 14 条边全部创建失败（节点 DOM 在但边 DOM 0；measured 空是连锁反应，非根因）。

### 改动明细
| 改动对象 | 类型 | 影响范围 | 改动明细 | 原因 |
|---------|------|---------|---------|------|
| `console/src/pages/Agents/GraphNode.tsx` | 修复 | 图调试 reactflow 主图 | 节点加透明 `<Handle>`（`style:{opacity:0}` 隐藏连接圆点但保留 attach 能力）：普通节点左 target + 右 source Handle（LR 布局）；`__start__` 右 source Handle（只出边）；`__end__` 左 target Handle（只入边） | reactflow 11 自定义节点必须含 Handle 才能 attach edge（error #008） |

### 验证
- **vite dev + puppeteer（1920×1080）**：`edges 14/14 渲染`，reactflow `Couldn't create edge` 警告从 14 条降为 0；节点 measured 正常填充
- **前端 build**：`npm run build` exit 0
- **部署**：push `1f70936` → CI id=80 success（5 分钟）→ 三台容器全部 `1f70936`，stream 健康

### 影响范围总结
- **图调试页**：reactflow 图的节点连线恢复渲染（14 条边 dagre LR 布局）。节点视觉/高亮/联动对照表不变。
- **诊断方法**：用 puppeteer 抓 console 警告定位到 reactflow error #008（Handle 缺失），避免在 measured/容器高度/width-height 上误判（那些是连锁现象）。
