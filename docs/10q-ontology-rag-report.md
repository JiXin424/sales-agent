## 两条路径分工总结

| 维度 | Ontology（Neo4j 知识图谱） | RAG（pgvector 文档检索） |
|------|--------------------------|------------------------|
| **数据结构** | 结构化三元组（Entity→Fact→Evidence） | 文档 chunk（语义向量） |
| **数据量** | 300 Entities, ~2000 Facts | 1925 chunks, 62 documents |
| **强项** | 硬数字（14000影院、200+品牌、500城市...） | 政策细节（延期规则、定价策略、品牌区别） |
| **弱项** | 大量事实 value=None，抽取质量待提升 | 语义相似可能命中不精确文档 |
| **补充关系** | 提供"骨架"（结构化的产品/品牌/数字） | 提供"血肉"（FAQ、政策解释、对比分析） |
| **典型贡献** | Q01影院数、Q03包邮事实、Q04福利卡定义 | Q04延期政策细节、Q06品牌数量差异、Q10零溢价 |

**结论**：Ontology + RAG 互补效果显著。Ontology 擅长的结构化事实为 LLM 提供精确数字约束，RAG 擅长的非结构化文本补充政策解释和对比细节。两者并行后 LLM 能综合两类信息给出完整答案。
