# 2026-06-25 升级日志

## Neo4j Ontology Knowledge Engine（本体知识引擎）

### 概述
新增 Neo4j 本体知识引擎（`KNOWLEDGE_ENGINE=ontology_neo4j`），用可审计的
Entity / Fact / Evidence 知识图谱替代传统 Markdown chunk RAG。首版采用图检索 + 保守
向量回退，支持入库可视化与高风险事实人工复核，并保持 chat 输出与现有 `summary` /
`sections` 响应格式兼容。传统 chunk RAG 路径保持不变，可通过环境变量切换。

---

### 改动明细

#### 1. Ontology 核心包
- **改动对象**：`sales_agent.ontology` 包（schemas / canonicalizer / neo4j_client / schema /
  repository / extractor / conflict / ingestion_service / retrieval_service / answer_service）
- **类型**：新功能
- **影响范围**：知识引擎核心逻辑
- **改动明细**：
  - `schemas` / `canonicalizer`：本体 schema 定义与实体规范化。
  - `neo4j_client`：Neo4j 异步连接管理。
  - `schema`：Cypher 约束 + 向量索引初始化。
  - `repository`：图谱 CRUD（Entity / Fact / Evidence / SourceDocument）。
  - `extractor`：LLM 抽取 + JSON 解析。
  - `conflict`：确定性风险分级。
  - `ingestion_service`：入库编排。
  - `retrieval_service`：图检索 + 保守向量回退。
  - `answer_service`：保持 `summary` / `sections` 输出格式。
- **原因**：建立可审计、可解释的知识图谱替代黑盒 chunk 检索。

#### 2. 配置与依赖
- **改动对象**：`OntologyConfig` / `Neo4jConfig`（Settings + 环境变量覆盖）、`pyproject.toml`
- **类型**：配置
- **影响范围**：应用配置
- **改动明细**：新增 `KNOWLEDGE_ENGINE`、`NEO4J_URI/USER/PASSWORD/DATABASE/VISUAL_URL`、
  `ONTOLOGY_VECTOR_FALLBACK` 等环境变量；新增 `neo4j>=5.23.0` 依赖。
- **原因**：支持引擎切换与 Neo4j 连接配置。

#### 3. ChatPipeline 检索路由
- **改动对象**：`services/chat_pipeline.py`
- **类型**：架构
- **影响范围**：聊天主链路
- **改动明细**：当 `knowledge_engine=ontology_neo4j` 且需要检索时，改走 `OntologyAnswerService`
  （引入 `skip_generation` 标志）；传统 chunk RAG 路径保持不变。
- **原因**：在不动 legacy 路径的前提下接入新引擎。

#### 4. IngestionJob 模型 + Alembic migration
- **改动对象**：`models/ingestion_job.py`、`migrations/versions/0003_ontology_neo4j_metadata.py`
- **类型**：数据模型 / 数据库变更
- **影响范围**：入库任务表
- **改动明细**：IngestionJob 新增 `agent_id` / `engine` / `stage` 列及实体 / 事实 / 冲突统计列；
  Alembic migration `0003_ontology_neo4j_metadata` 提供 upgrade / downgrade。
- **原因**：记录本体入库任务的图谱统计与阶段。

#### 5. Ontology API 路由
- **改动对象**：`api/routes/ontology.py`、`api/routes/health.py`、`main.py`
- **类型**：API
- **影响范围**：HTTP 接口
- **改动明细**：新增 `/agents/{id}/ontology/{status,ingest,jobs}`；`/ready` 在
  `ontology_neo4j` 模式下附带 Neo4j 就绪信息（`legacy_rag` 行为不变）。
- **原因**：暴露本体状态、入库触发与任务查询接口。

#### 6. 前端 Agents 知识页
- **改动对象**：`console/src/api/{types,knowledge}.ts`、`console/src/pages/Agents/AgentKnowledgePage.tsx`
- **类型**：前端
- **影响范围**：管理控制台
- **改动明细**：新增入库面板（状态 / 任务 / 冲突可视化 + 启动入库）；新增 ontology 状态与任务
  类型及 3 个 API wrapper。
- **原因**：让运营可视化本体入库进度与冲突。

#### 7. 测试
- **改动对象**：`tests/unit/`（ontology 单元/集成）、`tests/integration/`（fake 端到端）
- **类型**：测试
- **影响范围**：测试套件
- **改动明细**：ontology 单元/集成测试 + fake 端到端（ingest → retrieve → answer）；
  全量回归通过（ontology 34 + 回归 72 + 前端 build ok）。
- **原因**：保证新引擎功能正确且不破坏既有行为。

---

### 原因总结
用可审计的 Entity / Fact / Evidence 知识图谱替代 chunk RAG，支持入库可视化、保守向量回退、
高风险事实人工复核，提升知识库的可解释性与可控性。

### 部署注意
- 启用前需部署 Neo4j（≥5.23）并配置 `NEO4J_*` 环境变量。
- 首次启用执行 `alembic upgrade head` 应用 migration 0003。
- 传统模式（`KNOWLEDGE_ENGINE=legacy_rag` 或不设置）行为完全不变。
